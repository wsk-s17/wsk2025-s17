# Описание конкурсного задания

## Республиканский конкурс профессионального мастерства WorldSkills Kazakhstan 2025

### Компетенция: Веб-технологии

---

## Введение

В этом документе кратко описывается конкурсное задание по компетенции **17 – Веб-технологии**.
Документ содержит только основную информацию.
Окончательный вариант конкурсного задания будет представлен на **C-2** всем экспертам с использованием этого документа в качестве руководства.

---

## Описание проекта и заданий

Конкурсное задание состоит из **6 модулей** по следующим темам:

* **Модуль A:** Static Website Design
* **Модуль B:** Dynamic Website with Server-Side Rendering
* **Модуль C:** REST API
* **Модуль D:** Interactive Frontend using an API
* **Модуль E:** Speed Tasks
* **Модуль F:** Collaborative Challenge

Каждый модуль будет подробно описан ниже.

## Модуль A — Static Website Design

### Введение

Модуль A сосредоточен на создании **статического веб-сайта**.
Задача — разработать сайт для определённого клиента.
Распределение баллов представлено в таблице в конце раздела.

### Описание проекта и задач

Участники должны:

* создать **адаптивный веб-сайт** для клиента;
* использовать различные типы медиа;
* соблюдать **стандарты доступности (Accessibility)**.

Клиент предоставит:

* структуру сайта,
* текстовый и графический контент.

**Адаптивность** будет оцениваться по трём разрешениям:
мобильное устройство, планшет и компьютер.

### Проверка

* Оценка проводится в **Google Chrome** и **Firefox**.
* **HTML** и **CSS** проверяются валидаторами **W3C**.

### Время соревнований

⏱ **3 часа**

---

## Модуль B — Dynamic Website with Server-Side Rendering

### Введение

Модуль B посвящён созданию **динамического веб-сайта с серверным рендерингом**.

### Описание проекта и задач

Участники должны:

* разработать сайт для управления аккаунтами пользователей;
* реализовать **аутентификацию и безопасность (OWASP)**;
* спроектировать **структуру базы данных**;
* использовать **серверный фреймворк** (PHP, Node.js и др.);
* при необходимости — подключить дополнительные фронтенд-библиотеки,
  при этом рендеринг должен выполняться **на стороне сервера**.

### Проверка

Оценка проводится в **Google Chrome** и **Firefox**.

### Время соревнований

⏱ **3 часа**

---

## Модуль C — REST API

### Введение

Модуль B направлен на разработку **REST API** для **SPA-приложения** (модуль D).

### Описание проекта и задач

Участники должны:

* реализовать **полноценный RESTful API**;
* использовать базу данных **MySQL**;
* строго соблюдать спецификацию клиента:

  * конечные точки (endpoints),
  * структура запросов и ответов,
  * коды состояния HTTP.

### Проверка

Оценка проводится с использованием автотестов и/или **Insomnia** (последняя стабильная версия).

### Время соревнований

⏱ **3 часа**

---

## Модуль D — Interactive Frontend Using an API

### Введение

Модуль D направлен на разработку **SPA-приложения**, использующего API (из модуля С).

### Описание проекта и задач

Участники должны:

* создать **SPA веб-приложение**, взаимодействующее с RESTful API;
* реализовать:

  * авторизацию и вход в систему;
  * загрузку изображений;
  * интерфейс, соответствующий стандартам доступности;
* при необходимости можно использовать готовый API для завершения задания.

### Проверка

Оценка проводится в **Google Chrome** и **Firefox**.

### Время соревнований

⏱ **4 часа**

---

## Модуль E — Speed Tasks

### Введение

Модуль E состоит из **мини-заданий** по темам веб-разработки.

### Описание проекта и задач

Задачи могут включать:

* реализацию JS-функций по документации;
* создание и стилизацию UI-элементов;
* разработку интерактивных функций на JavaScript.

Задания делятся по уровням:

* **Лёгкие**
* **Средние**
* **Сложные**

Каждый уровень имеет разное количество очков.

### Проверка

* Проверка в **Google Chrome** и **Firefox**;
* **HTML** и **CSS** валидируются с помощью **W3C**.

### Время соревнований

⏱ **3 часа**

---

## Модуль F — Collaborative Challenge

### Введение

Модуль F направлен на **командную работу** и **креативность участников**.

### Описание проекта и задач

Участники объединяются в группы и:

* создают **небольшой веб-продукт или веб-опыт**;
* готовят **презентацию**, включающую:

  * описание идеи,
  * демонстрацию (демо),
  * объяснение концепции и процесса работы.

### Проверка

Критерии оценки:

* участие и взаимодействие в команде;
* наличие проработанной идеи;
* качество и профессионализм презентации.

### Время соревнований

⏱ **2 часа**
