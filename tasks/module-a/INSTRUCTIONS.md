# Module A - Static Website Design

В этом модуле вам необходимо разработать рекламный лендинг для онлайн-сервиса объявлений. Это платформа, где пользователи размещают и просматривают объявления о продаже коллекционных и тематических товаров: конструкторы LEGO, настольные игры, комиксы, фигурки, сувениры и прочее.

Ваша задача – реализовать современный лендинг, который расскажет о возможностях сервиса и его преимуществах для разных типов пользователей.

## Competitor Information

Лендинг должен состоять из четырех логически связанных страниц. Каждая страница должна быть доступна по навигации, иметь единый стиль и корректно отображаться на устройствах разных размеров.

Пожалуйста, учитывайте на страницах SEO-оптимизацию и необходимые тэги. На веб-странице необходимо настроить **OpenGraph**.

Приложите файл **README.md** с инструкцией по запуску проекта. В данном файле опишите процесс изменения ссылки на кнопках `Опубликовать` и `Посмотреть`.

## Website Requirements

- Не допускается использование фреймворков
- Адаптивная верстка: корректное отображение на устройствах с разным разрешением
- Единая структура навигации на всех страницах
- Семантическая и чистая разметка
- Наличие микро-анимаций эффектов при наведении на кнопки и ссылки
- Качественная типографика и читаемый текст
- Учитывайте правильность типов на полях и корректные ссылки

### Home Page

Необходимо отразить следующую информацию:
- Промо-блок с названием проекта, кратким описанием и Call-to-action
- Информация о проекте: цели, аудитория, особенности платформы
- Категории товаров (`media-files/texts/category.txt`)
- Преимущества сервиса
- Call-to-action блок для привлечения пользователя
- Блок отзывов пользователей

### For Seller Page

Необходимо отразить следующую информацию:
- Инструкция по размещению объявлений. Вам необходимо расписать процесс по размещению объявлений, рассказывая про процесс и его легкость. Storytelling должен быть увлекательным и отображать ценности сервиса (`media-files/texts/seller-story.txt`)
- Преимущества для продавца (`media-files/texts/seller.txt`)
- Секция FAQ (`media-files/texts/seller-faq.txt`)

### For Seller - Advantages Page

Необходимо отразить следующую информацию:
- Прайс-лист (`media-files/texts/seller-price-list.txt`). Вы должны дать возможность выбрать пользователю выбрать количество дней для расчета стоимости каждой услуги. Пользователь должен выбирать количество дней через тип поля `range`. Цены и количество дней описаны в файле.
- Преимущества для продавца (`media-files/texts/seller-price-list-advantages.txt`)
- Секция FAQ (`media-files/texts/seller-price-list-faq.txt`)

### For Buyer Page

Необходимо отразить следующую информацию:
- Инструкция по покупке в виде пошагового описания. Вам необходимо расписать процесс по размещению объявлений, рассказывая про процесс и его легкость. Storytelling должен быть увлекательным и отображать ценности сервиса (`media-files/texts/buyer-story.txt`)
- Преимущества для покупателя (`media-files/texts/buyer.txt`);
- Секция FAQ (`media-files/texts/buyer-faq.txt`)

### Contacts

Необходимо отразить следующую информацию:
- Форма обратной связи (имя, email, сообщение)
- Контактная информация: электронная почта, часы работы
- Ссылки на социальные сети
- Карта (`media-files/map.png`)

Информация расположена в файле `media-files/texts/contacts.txt`.

### Policy

Необходимо разместить текст политики конфиденциальности, предоставленный в файле `media-files/texts/policy.txt`, на отдельной странице.

Текст должен быть оформлен с использованием HTML-элементов, обеспечивающих его читаемость: заголовки, абзацы, списки и другие подходящие элементы разметки.

## Global Elements

На каждой странице должен быть размещен **header** с следующими элементами:
- Логотип
- Навигация
- Кнопки `Publish` и `View`

Данные кнопки в данный момент ведут на страницы `For Seller` и `For Buyer`, но должна быть возможность легко изменить ссылку на кнопки.

Предусмотрите 2 разных стиля и микро-анимации для данных кнопок, они должны изменяться по атрибуту data-type.

В блоке навигации должны быть следующие пункты меню:
- Home
- For Seller
- - Advantages
- - Paid Services
- - FAQ
- For Buyer
- - Advantages
- - FAQ

У каждого пункта меню (кроме `Home`) есть второй уровень меню. Он должен появляться по наведению в виде отдельного блока.

Учтите, что для отображения второго уровня меню **не должен использоваться JavaScript**.

На каждой странице реализуйте футер с навигацией и ссылками на другие страницы.
