# Module D - Interactive Frontend using an API

В этом модуле необходимо разработать клиентскую часть (SPA) для онлайн-сервиса объявлений. Это веб-интерфейс, взаимодействующий с REST API, реализованным в модуле C.

## Competitor Information

Приложение должно обеспечивать работу с API: авторизацию, управление объявлениями, просмотр конкретных объявлений, а также реализацию особенностей, связанных с ролями и статусами.

Вам предоставлена спецификация **OpenAPI** по которому работает API, пожалуйста, используйте **готовое решение модуля C** для разработки модуля.

API доступен по адресу: `https://module-c-api.YYYYYY.wsk.com`, где YYYYYY - ваш уникальный **subdomain**.

- Используйте предоставленный фреймворк из вкладки `templates` в GIT
- Интерфейс должен быть адаптивным: корректно отображаться на разных устройствах
- Обязательно реализовать навигацию между страницами (например, с использованием client-side routing)
- Предусмотреть обработку ошибок: неверные данные, отсутствие доступа, ошибки сервера и т.д.
- Обеспечьте пользовательский опыт: загрузка данных, спиннеры, уведомления, подтверждения действий

## Website Requirements

Необходимо реализовать следующие функции:
- Регистрация пользователя. Форма регистрации с полями: email, телефон, пароль. Валидация должна быть на клиентской части, все поля обязательные. После успешной регистрации пользователь автоматически входит в систему.
- Авторизация. Возможность входа по email или телефону + пароль. После входа пользователь получает токен, который хранится на клиенте. При каждом запросе токен передается в заголовке Authorization.
- Редактирование профиля. Пользователь может изменять свой E-mail, номер телефона и имя. Учтите соответствующие правила валидации.
- Выход. Пользователь должен иметь возможность выйти из системы. При этом токен удаляется из браузера.

### Adverts

Можно создавать объявления. Форма создания объявления должна включать: заголовок, текст, цену, категорию, загрузку изображений.

После создания объявление имеет статус `draft`. Пользователь может отправить его на модерацию (статус `moderation`).

Можно редактировать объявления. Редактирование возможно только в статусе `draft`. Форма редактирования аналогична форме создания.

Пользователь также может перевести объявление в статус `archived`.

Можно удалить объявления. Удаление доступно только автору в статусе `draft` или `archived`.

Можно просмотривать свои объявления. Страница `Мои объявления` должна содержать список объявлений пользователя. Объявления можно фильтровать по статусам.

### Home Page

На странице отображается список объявлений. Страница не требует авторизации.

Отображение всех опубликованных (published) объявлений. Для каждого объявления отображаются: название, краткое описание, цена, категория, дата.

Возможность отфильтровать объявления по выбранной категории, цене и по ключевым словам. Цену можно фильтровать от минимальной до максимальной, оба поля не обязательные.

### Advert Details Page

Можно просмотривать детали объявления. Отображаются: название, описание, цена, категория, изображения, дата
публикации, имя автора.

Изображения нужно отображать в виде карусели: одно изображение по центру, и под ним все остальные изображения. Учтите стандартные требования к карусели.

Можно просмотривать телефон продавца. Кнопка `Показать телефон`. По нажатию появляется номер телефона. До этого он скрыт. Когда номер скрыт, должно отображаться только первые 4 цифры номера, а остальное скрыто за кнопкой.

Рядом с объявлением нужно отображать кнопку для покупки платных услуг, а
также список уже купленных услуг, вместе с датой их истечения.

### Purchase Page For Advantage Services

Для каждого объявления можно купить платные услуги.

Предложите уникальный интерфейс для покупки платных услуг, и учтите модульность данной системы.

Если услуга не куплена, то пользователь может ее купить на 3/7 дней.

Если услуга уже куплена, то он сможет ее продлить после истечения срока.
