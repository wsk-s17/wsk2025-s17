# Module B - Dynamic website with server-side rendering

Вам необходимо разработать админ-панель для сервиса объявлений - гик-барахолка. Это будет интерфейс модераторов платформы, где пользователи могут размещать и просматривать объявления о продаже конструкторов лего, настольных игр, книг, сувениров и т.д.

В данном модуле нужно будет реализовать управление объявлениями и их категориями.

Внешний вид на ваше усмотрение, но UX будет оцениваться.

## Website Requirements

### Auth

Авторизоваться в админ-панель могут только пользователи с ролью `moderator`.

Форма входа содержит поля ввода логина и пароля. В качестве логина можно ввести **номер телефона** или **email**.

В случае успешной авторизации пользователя редиректит на главную страницу админ-панели.
В случае ввода неправильных данных пользователю отображается сообщение.

### Header

Хедер должен содержать кликабельные ссылки на главную, страницу списка категорий, списка объявлений, списка пользователей.

### Home Page

Страница содержит следующие данные:
- Имя текущего авторизованного пользователя
- Количество объявлений в различных статусах на текущий момент
- Количество пользователей
- Топ 10 объявлений по просмотрам (только в статусе `published`)

### Categories Page

Страница содержит список категорий (ID, название, количество объявлений в статусе `published`) и интерфейс для добавления, редактирования и удаления категорий (может быть как в виде модальных окон, так и отдельных страниц). Учтите, что удалить категорию, к которой привязаны объявления в любом статусе, нельзя.

### Users Page

Страница содержит список пользователей (ID, имя, телефон, email, количество объявлений в статусе `published`).
На странице должно быть одно текстовое поле для поиска одновременно по ID, номеру телефона и email (точное совпадение).

### Adverts Page

Страница содержит список объявлений и фильтры по статусу, категорию, тексту. Фильтр по тексту должен искать по заголовку, тексту объявления, названию категории и имени пользователя-автора.
В списке объявлений должна отображаться пометка о подключенных платных услугах, если есть.

### Advert Details Page

Страница содержит следующие данные:
- ID, заголовок, текст, цена, количество просмотров, название категории, статус, список фото (в виде плитки)
- ID, имя, телефон и email пользователя-автора
- Интерфейс смены статуса
- Данные о подключенных платных услугах, в том числе истекших (тип и дата подключения, срок действия)
- Должна быть возможность подключить или отключить платную услугу объявлению

Возможные переходы между статусами:
- `moderation` → `published`
- `moderation` → `declined`
- `published` → `declined`

### Export adverts list

На странице списка объявлений должна быть кнопка экспорта, по нажатию на которую список объявлений выгружается в файл `export_adverts.csv` со следующими столбцами:
- ID
- Заголовок
- Название категории
- Цена
- Номер телефона автора
- Email автора
- Текст
- Дата размещения

Если на странице выбраны **фильтры**, то в выгрузку должен попасть только отфильтрованный список записей.
